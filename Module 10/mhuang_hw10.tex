data = c(1, 2, 3.5, 4, 7.3, 8.6, 12.4, 13.8, 18.1)
\end{spverbatim}

\subsection*{(a)} 

We use algorithm for calculating the bootstrap variance directly; for each bootstrap iteration: \\
- Resample \(X\) with replacement to generate the bootstramp sample \\
- Compute the trimmed mean based on the sample \\
Finally, take the sample variance of all the bootstrap trimmed means.

\begin{spverbatim}
> data = c(1, 2, 3.5, 4, 7.3, 8.6, 12.4, 13.8, 18.1)
> 
> trimmed_mean = function(data) {
+   sorted = sort(data)
+   trimmed = sorted[3:(length(sorted) - 2)]
+   return(mean(trimmed))
+ }
> 
> bootstrap_var = function(data, B) {
+   trimmed_means = rep(0, B)
+   for (i in 1:B) {
+     x_star = sample(data, length(data), replace = TRUE)
+     trimmed_means[i] = trimmed_mean(x_star)
+   }
+   return(var(trimmed_means))
+ }
> 
> B_values = c(25, 100, 200, 500, 1000, 2000)
> for (B in B_values) {
+   print(paste("B =", B))
+   print(bootstrap_var(data, B))
+ }
[1] "B = 25"
[1] 3.819356
[1] "B = 100"
[1] 6.256274

[1] 5.104036
[1] "B = 1000"
[1] 5.835679

\end{spverbatim}

From these values, we see the bootstrapped variance seemingly converging over time to be about 6.

\section*{2.}

We are given 
\begin{spverbatim}
Survival = c(25, 42, 45, 46, 51, 103, 124, 146, 340, 396, 412, 879, 1112)
\end{spverbatim}

\subsection*{(a)} 

\begin{spverbatim}
> survival = c(25, 42, 45, 46, 51, 103, 124, 146, 340, 396, 412, 879, 1112)
> 
> B = 1000
> 
> log_survival = log(survival)
> boot_means_log = rep(0, B)
> 
> for (i in 1:B) {
+   sample_log_survival = sample(log_survival, length(log_survival), replace = TRUE)
+   boot_means_log[i] = mean(sample_log_survival)
+ }
> 
> # Taking the 95% CI means 2.5% on either side
> ci_log = quantile(boot_means_log, c(0.025, 0.975))
> # Exponentiate after taking log
> ci_original_scale = exp(ci_log)
> 
> print("95% CI for log/exponentiated bootstrap:")
[1] "95% CI for log/exponentiated bootstrap:"
> print(ci_original_scale)
     2.5%     97.5% 
 74.85874 277.46767 
\end{spverbatim}

\subsection*{(b)}

\begin{spverbatim}
> boot_means_original = rep(0, B)
> 
> for (i in 1:B) {
+   sample_survival = sample(survival, length(survival), replace = TRUE)
+   boot_means_original[i] = mean(sample_survival)
+ }
> 
> ci_raw = quantile(boot_means_original, c(0.025, 0.975))
> 
> print("95% CI for original scale bootstrap:")
[1] "95% CI for original scale bootstrap:"
> print(ci_raw)
    2.5%    97.5% 
130.7442 485.9481 
\end{spverbatim}

Using the bootstrap on the logged data, the 95\% confidence interval for mean survival time was \([74.85874, 277.46767]\) compared to \([130.7442, 485.9481]\) using the normal data. We see that the logged data confidence interval has a much smaller range and is also lower. This suggests that it is more accurate for this dataset, which we note has a considerable right skew, with very few data points on the higher end compared to the lower end. So the logged approach seems to work better for data with the more extreme outliers and skewness. The transformation seems to work better compared to the simple arithmetic mean, which is likely more affected by the outlier points.
